1. INTRODUCTION                                             *copane-intro*

copane is an AI-powered coding assistant that runs in a tmux pane and
integrates with Vim/Neovim. It allows you to send code snippets for review,
refactoring, explanation, and test generation directly from your editor.

Features:
  - AI-powered code analysis and suggestions
  - Tmux integration for persistent assistant session
  - Multiple AI model support (DeepSeek, OpenAI, Ollama)
  - Custom model endpoints via model_config.json
  - File inclusion in queries via @filename
  - Professional terminal interface with syntax highlighting
  - Conversation history across buffer splits
  - File completion (Tab) and multi-line input
  - AI agent tools (read files, run commands, grep, list files)

2. INSTALLATION                                         *copane-installation*

Requirements:
  - Python 3.12+
  - tmux
  - Vim 8+ or Neovim 0.5+
  - pip or uv (package manager)

2.1 Using vim-plug ~

Add to your ~/.vimrc (or ~/.config/nvim/init.vim for Neovim):
>
    Plug 'MostafaKashwaa/copane'
<

Then run:
>
    :PlugInstall
    :CopaneOpen
<

The Python virtual environment is set up automatically on first use.
If the automatic setup fails, run:
>
    :CopaneSetupPython
<

2.2 Manual Installation ~

Clone the repository:
>
    git clone https://github.com/MostafaKashwaa/copane.git \
      ~/.vim/plugged/copane
<

For Neovim:
>
    git clone https://github.com/MostafaKashwaa/copane.git \
      ~/.local/share/nvim/site/pack/plugins/start/copane
<

Set up the Python environment:
>
    bash ~/.vim/plugged/copane/setup_python.sh
<

2.3 Standalone Installation (without a plugin manager) ~

    bash <(curl -sSL https://raw.githubusercontent.com/MostafaKashwaa/copane/main/install.sh)

This installs copane to ~/.copane/ and adds the `copane` command to
~/.local/bin/. Make sure that directory is on your PATH.

2.4 Running from the terminal ~

Once installed, you can run copane directly:
>
    copane
<

Or using the virtual environment directly:
>
    ~/.vim/plugged/copane/python/.venv/bin/python3 -m copane.app
<

3. CONFIGURATION                                       *copane-configuration*

3.1 API Keys ~

Create the environment file:
>
    cp .env.example ~/.copane.env
    vim ~/.copane.env
<

Add your API keys. Supported variables:
>
    DEEPSEEK_API_KEY=sk-...
    OPENAI_API_KEY=sk-...
    GROQ_API_KEY=gsk_your-groq-api-key
    OPENROUTER_API_KEY=sk-or-v1-your-openrouter-api-key
    OLLAMA_BASE_URL=http://localhost:11434


You can also set any other environment variables your custom endpoints
require. The file is loaded via python-dotenv.

The default location is ~/.copane.env. Override it with: >
    let g:copane_env_file = '/path/to/your/.env'
<

3.2 LangSmith Tracing (Optional) ~

The plugin uses langsmith to trace tool calls and LLM responses. To enable,
set the standard LangSmith environment variables in your .copane.env: >
    LANGCHAIN_TRACING_V2=true
    LANGCHAIN_API_KEY=lsv2_pt_...
    LANGCHAIN_PROJECT=copane
<

See the LangSmith documentation at
https://docs.smith.langchain.com for setup details.

3.3 Model Configuration ~

Models and endpoints are stored in:
  ~/.config/tmux-agent/model_config.json

This file is auto-generated on first run. Edit it to:
  - Add custom API endpoints (local vLLM, Groq, etc.)
  - Change the default/active model
  - Remove models you don't use

Open it quickly: >
    :CopaneEditConfig
<

Default models include:
  - deepseek-chat (DeepSeek Chat — default)
  - gpt-4o (OpenAI GPT-4o)
  - local-ollama (Local Ollama with gemma4:26b)

3.4 Vim Configuration ~

All configuration variables use the g:copane_ prefix.

General settings: >
    " Python executable for the copane app
    let g:copane_python_path = 'python3'

    " Name of the tmux pane (used to identify the copane pane)
    let g:copane_tmux_pane_name = 'copane'

    " Environment file for API keys
    let g:copane_env_file = expand('~/.copane.env')

    " Virtual environment directory
    let g:copane_venv_dir = expand('~/.vim/plugged/copane/python/.venv')

    " Default model on startup
    let g:copane_default_model = 'deepseek-chat'

    " Tmux target for the pane (empty = current session)
    let g:copane_tmux_target = ''
<

Tmux pane appearance: >
    " Size of the split pane (number or percentage like '33%')
    let g:copane_split_size = '33%'

    " Direction of the split:
    "   'vertical'   — split horizontally (pane on the right)
    "   'horizontal' — split vertically (pane below)
    "   'below'      — same as 'horizontal'
    let g:copane_split_direction = 'vertical'

    " Pane scope:
    "   'session' — one pane for the entire tmux session
    "   'window'  — one pane per tmux window
    let g:copane_pane_scope = 'session'

    " Custom start command (overrides the auto-detected command)
    let g:copane_start_command = ''

    " Automatically open copane on startup
    let g:copane_auto_open = 0
<

Keybinding and UI: >
    " Prefix for all default mappings (default: <leader>t)
    let g:copane_mapping_prefix = '<leader>t'

    " Disable default mappings entirely
    let g:copane_no_mappings = 0

    " Show the startup banner (1) or skip it (0)
    let g:copane_show_banner = 1

    " Debug mode (verbose output)
    let g:copane_debug = 0

    " Hide the "Ready!" suggestion message on load
    let g:copane_no_suggestions = 0
<

Feature flags: >
    " Load filetype-specific copane plugins (ftplugin/)
    let g:copane_enable_ftplugin = 1

    " Enable Neovim async remote plugin integration
    let g:copane_enable_neovim_async = 1
<

3.5 tmux Configuration ~

For a convenient keybinding to open copane, add to ~/.tmux.conf:
>
    bind-key a split-window -h "copane"
<

4. COMMANDS                                               *copane-commands*

4.1 Pane Commands ~

:CopaneOpen
    Open (create or focus) the copane tmux pane. If a copane pane
    already exists in the current tmux session, it will be moved to
    the current window and selected. Otherwise, a new pane is created
    running the copane Python app.

:CopaneClose
    Close the copane tmux pane. Clears the stored pane ID.

:CopaneToggle
    Toggle focus between the copane pane and the previously selected pane.

4.2 Send Commands ~

:CopaneSend [text]
    Send the current buffer content (with file path) to copane.
    If optional text is provided, send that instead.

:'<,'>CopaneSend
    Send the visual selection (with file path and line numbers) to
    copane. Works in visual mode with the range marker.

4.3 Model Commands ~

:CopaneModelInfo
    Show information about the currently selected model in the pane.

:CopaneSwitchModel {key}
    Switch to a different model (e.g. deepseek-chat, gpt-4o,
    local-ollama).

:CopaneListModels
    List all available models with their status and type.

4.4 Setup Commands ~

:CopaneSetupPython
    Run the Python setup script (setup_python.sh) to create the
    virtual environment and install dependencies in one step.
    Runs automatically on first :CopaneOpen if the venv doesn't exist.

:CopaneEditSecrets
    Open the API keys environment file (~/.copane.env by default).
    If the file doesn't exist, creates it from a template.

:CopaneEditConfig
    Open the model configuration file
    (~/.config/tmux-agent/model_config.json).
    Creates it with default models if it doesn't exist.

4.5 Utility Commands ~

:CopaneHelp
    Show a quick-reference help summary in the Vim command line.

:CopaneDebug
    Show diagnostic information: stored pane ID, pane status,
    session scope, venv path, env file, and current options.

:CopaneClearHistory
    Clear the conversation history inside the copane pane.

4.6 Legacy Commands ~

The following aliases exist for backward compatibility:
>
    TmuxAgentOpen       (same as :CopaneOpen)
    TmuxAgentClose      (same as :CopaneClose)
    TmuxAgentSend       (same as :CopaneSend)
    TmuxAgentSendVisual (same as :CopaneSendVisual)
<

5. MAPPINGS                                               *copane-mappings*

All mappings use the |g:copane_mapping_prefix| variable (default
|<leader>t|). The table below shows them with the default prefix.

To use a different prefix: >
    let g:copane_mapping_prefix = '<leader>x'
<

Mapping              Mode   Action~
<Leader>to           normal Open copane pane (:CopaneOpen)
<Leader>tc           normal Close copane pane (:CopaneClose)
<Leader>tt           normal Toggle copane pane (:CopaneToggle)
<Leader>ts           normal Send current buffer (:CopaneSend)
<Leader>ts           visual Send visual selection (:CopaneSendVisual)
<Leader>tm           normal Show current model info (:CopaneModelInfo)
<Leader>tM           normal List available models (:CopaneListModels)
<Leader>te           normal Edit secrets file (:CopaneEditSecrets)
<Leader>tE           normal Edit model config (:CopaneEditConfig)
<Leader>th           normal Show help (:CopaneHelp)

Filetype-specific mappings (ftplugin/) add:
<Leader>ta           normal Send buffer (Python/JS buffers)
<Leader>ta           visual Send selection (Python/JS buffers)
<Leader>te           normal Explain code (Python/JS buffers)
<Leader>te           visual Explain selection (Python/JS buffers)
<Leader>tt           normal Write tests (Python/JS buffers)
<Leader>tt           visual Write tests for selection (Python/JS buffers)

To disable default mappings: >
    let g:copane_no_mappings = 1
<

Then define your own: >
    nmap <leader>cc :CopaneSend<CR>
    nmap <leader>cs :CopaneEditSecrets<CR>
<

6. USAGE                                                     *copane-usage*

6.1 Starting copane ~

Run directly in a terminal:
>
    copane
<

Or from Vim:
>
    :CopaneOpen
<

The plugin will:
  1. Check for an existing copane pane (by stored pane ID)
  2. Move it to the current window if it exists in the session
  3. Create a new one if it doesn't exist (splitting the current window)
  4. Run the Python app inside the pane

6.2 Command-line Flags ~
>
    # Interactive mode (default)
    copane

    # Quick actions
    copane --mode explain --file main.py
    copane --mode test --text "def add(a,b): return a+b"
    copane --mode review --file main.py
    copane --mode refactor --file src/utils.py

    # Model management
    copane --list-models
    copane --switch gpt-4o
    copane --model-info

    # Use a specific env file
    copane --env-file ~/.copane.env

    # Skip the banner (useful for embedding)
    copane --no-banner
<

6.3 Interactive Commands ~

While in the copane terminal, use these special commands:

    /models               List all available models
    /switch <model>       Switch to a different model
    /modelinfo            Show current model details
    /clear                Clear conversation history
    /help                 Show help
    exit / quit / q       Exit copane

Keyboard shortcuts:
    Ctrl-C                Cancel current response (asks for confirmation)
    Tab                   Auto-complete file paths (after @)
    Ctrl-J                Submit multi-line input

6.4 File Inclusion ~

Reference files in your query by prefixing with @:
>
    Please review @pkg/main.py for security issues
    Explain the @src/utils.py::parse_config function
<

Use Tab to auto-complete file paths after typing @.

6.5 AI Agent Tools ~

The copane agent has access to the following tools and will use them
proactively to understand your codebase:

  - read_file(path, start_line, end_line)
  - run_command(cmd)
  - grep_files(pattern, path, file_glob)
  - list_files(path, depth)
  - write_file(path, content)     — requires confirmation
  - get_current_dir()

7. TROUBLESHOOTING                                 *copane-troubleshooting*

7.1 "Python dependencies not installed" ~

Run the setup command:
>
    :CopaneSetupPython
<

Or manually:
>
    cd ~/.vim/plugged/copane
    bash setup_python.sh
<

7.2 "API key not found" ~

Check your environment file:
>
    :CopaneEditSecrets
<

Or from the shell:
>
    cat ~/.copane.env
<

Also verify the file is loaded: >
    :CopaneDebug
<

Look for the "Env file" line.

7.3 "copane command not found" ~

Ensure the virtual environment's bin directory is on your PATH:
>
    export PATH="$PATH:$HOME/.vim/plugged/copane/python/.venv/bin"
<

Or use the full path:
>
    ~/.vim/plugged/copane/python/.venv/bin/python3 -m copane.app
<

7.4 Tmux pane not appearing ~

Check that tmux is running:
>
    tmux list-sessions
<

Verify the pane name with debug info:
>
    :CopaneDebug
<

Check for "Stored pane ID" and "Pane exists" lines.

7.5 Vim mappings not working ~

Verify the plugin is loaded:
>
    :echo exists('g:loaded_copane')
<

(Should return 1.)

Check your leader key:
>
    :echo g:copane_mapping_prefix
<

7.6 Model errors ~

If you get "model not found" or API errors:
>
    :CopaneEditConfig
<

Verify the provider URL, API key env variable name, and model name.
For Ollama, ensure the service is running:
>
    curl http://localhost:11434/api/tags
<

7.7 Debug Mode ~

Enable verbose logging: >
    let g:copane_debug = 1
<

Then use:
>
    :CopaneDebug
<

From the python directory, run:

8.4 Architecture ~

The copane Python application runs as a persistent process inside a tmux
pane. The Vim plugin communicates with it by sending text directly to the
pane (via tmux send-keys).

Key design decisions:
  - Python app is self-contained and can run without Vim
  - Vim-side code is minimal (pane management + keybindings)
  - All AI logic, model config, and UI lives in Python
  - Environment file path is passed via --env-file argument
  - Model configuration lives in ~/.config/tmux-agent/ (persistent)
  - Agent has tool access (read files, run commands, grep, etc.)
  - Neovim has optional async remote plugin support (rplugin/)
  - Tool calls and LLM responses are traced via langsmith
